# Hetzner VPS Provisioning & Configuration

Complete infrastructure-as-code solution for provisioning and configuring Hetzner Cloud VPS servers using Terraform and Ansible.

## Quick Start

### 1. Clone Repository

```bash
cd ~/projects
git clone https://github.com/tgunawan/hetzner-vps-provision.git
cd hetzner-vps-provision
```

### 2. Configure Terraform Variables

```bash
cd terraform

# Create terraform.tfvars
cat > terraform.tfvars <<EOF
# Hetzner API Token
hcloud_token = "your_hetzner_api_token_here"

# Project Configuration
project_name = "my-project"
environment  = "production"

# SSH Configuration
ssh_key_name         = "terraform-key"
ssh_public_key_path  = "~/.ssh/id_ed25519.pub"

# Server Configuration
server_count    = 1
server_type     = "cpx21"     # 3 vCPU, 4GB RAM, 80GB disk
server_image    = "ubuntu-22.04"
server_location = "nbg1"      # Nuremberg, Germany

# Security
ssh_allowed_ips = ["your_ip_here/32"]  # Replace with your IP

# Optional Features
enable_private_network    = false
enable_additional_storage = false
enable_load_balancer      = false
EOF
```

### 3. Provision Infrastructure

```bash
# Initialize Terraform
terraform init

# Preview changes
terraform plan

# Apply configuration
terraform apply

# Save outputs
terraform output -raw ansible_inventory_yaml > ../ansible/inventory.yml
terraform output -raw summary
```

### 4. Configure Servers with Ansible

```bash
cd ../ansible

# Test connection
ansible all -i inventory.yml -m ping

# Run configuration playbook
ansible-playbook -i inventory.yml playbook.yml

# Optional: Deploy Dokploy
ansible-playbook -i inventory.yml playbook.yml -e "enable_dokploy=true dokploy_domain=yourdomain.com"
```

### 5. Verify Deployment

```bash
# SSH to server
ssh root@<server_ip>

# Check Docker
docker --version
docker ps

# Check firewall
sudo ufw status

# Check fail2ban
sudo fail2ban-client status sshd
```

## Advanced Usage

### Multiple Servers with Private Network

```hcl
# terraform.tfvars
server_count = 3
enable_private_network = true
private_network_ip_range = "10.0.0.0/16"
private_subnet_ip_range  = "10.0.1.0/24"
```

Servers will be assigned IPs: `10.0.1.10`, `10.0.1.11`, `10.0.1.12`

### High Availability with Load Balancer

```hcl
# terraform.tfvars
server_count = 2
enable_load_balancer = true
load_balancer_type   = "lb11"  # 5,000 connections
```

Access your application via the load balancer IP instead of individual server IPs.

### Additional Storage Volumes

```hcl
# terraform.tfvars
enable_additional_storage = true
storage_size_gb          = 100  # 100GB per server
```

Volumes are automatically mounted at `/mnt/HC_Volume_<id>`

### Custom Firewall Rules

```hcl
# terraform.tfvars
custom_tcp_ports = [
  {
    port        = "5432"  # PostgreSQL
    allowed_ips = ["10.0.0.0/8"]  # Private network only
  },
  {
    port        = "8080"  # Application
    allowed_ips = ["0.0.0.0/0"]  # Public
  }
]
```

## Security Best Practices

### ✅ Implemented

- **SSH Key-Only Authentication**: Password authentication disabled
- **UFW Firewall**: Default deny incoming, allow outgoing
- **Fail2ban**: Protects against brute-force SSH attacks
- **Automatic Security Updates**: Unattended upgrades enabled
- **Hardened SSH**: Limited authentication attempts, no root password login
- **IP Whitelisting**: SSH access restricted to specified IPs

### ⚠️ Recommended Additional Steps

1. **Change SSH Port**:

   ```bash
   sudo nano /etc/ssh/sshd_config
   # Change Port 22 to Port 2222
   sudo systemctl restart sshd
   sudo ufw allow 2222/tcp
   sudo ufw delete allow 22/tcp
   ```

1. **Enable Two-Factor Authentication**:

   ```bash
   sudo apt install libpam-google-authenticator
   google-authenticator
   ```

1. **Set Up Log Monitoring**:

   - Install Logwatch or Fail2ban email alerts
   - Configure centralized logging (Loki, ELK)

1. **Regular Backups**:

   - Enable Hetzner Cloud Backups (auto snapshots)
   - Or use restic/borg for custom backup solution

1. **SSL Certificates**:

   ```bash
   sudo apt install certbot python3-certbot-nginx
   sudo certbot --nginx -d yourdomain.com
   ```

## Troubleshooting

### Issue: Terraform Apply Fails with 401 Unauthorized

**Cause**: Invalid or expired Hetzner API token

**Solution**:

1. Generate new token in Hetzner Cloud Console
1. Update `hcloud_token` in `terraform.tfvars`
1. Run `terraform apply` again

### Issue: SSH Connection Refused

**Cause**: Firewall blocking connection or wrong IP

**Solution**:

```bash
# Check if your IP is allowed
terraform show | grep ssh_allowed_ips

# Update firewall rule
terraform apply -var='ssh_allowed_ips=["your_new_ip/32"]'
```

### Issue: Ansible Playbook Fails at Docker Installation

**Cause**: APT repository not accessible

**Solution**:

```bash
# SSH to server manually
ssh root@<server_ip>

# Update manually
apt-get update
apt-get install docker-ce -y

# Re-run playbook
ansible-playbook -i inventory.yml playbook.yml --start-at-task="Install Docker"
```

### Issue: Server Runs Out of Memory

**Cause**: No swap configured or insufficient RAM

**Solution**:

```bash
# Increase swap size
ansible-playbook -i inventory.yml playbook.yml -e "swap_size_mb=4096"

# Or upgrade server type
terraform apply -var='server_type=cpx31'
```

## Cost Optimization

### Reduce Costs

1. **Use smaller server types**: `cpx11` for dev/staging
1. **Disable load balancer**: For single-server setups
1. **Use snapshots instead of backups**: Manual instead of automatic
1. **Delete unused volumes**: Check with `terraform state list`

## Maintenance

### Update Server Software

```bash
# Via Ansible
ansible all -i inventory.yml -m apt -a "upgrade=dist" --become

# Or SSH manually
ssh root@<server_ip>
apt-get update && apt-get upgrade -y
```

### Scale Infrastructure

```bash
# Add more servers
terraform apply -var='server_count=3'

# Change server type
terraform apply -var='server_type=cpx31'
```

### Backup and Restore

```bash
# Create snapshot via Hetzner CLI
hcloud server create-image my-project-1 --type snapshot --description "Manual backup $(date +%Y%m%d)"

# Restore from snapshot
terraform apply -var='server_image=<snapshot_id>'
```

## Destroying Infrastructure

```bash
cd terraform

terraform plan -destroy
```
