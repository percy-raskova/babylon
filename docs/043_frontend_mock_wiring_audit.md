Scope: this report traces the actual data circulation used by the current frontend entrypoints in [../web/frontend/src/App.tsx](../web/frontend/src/App.tsx), [../web/frontend/src/main.tsx](../web/frontend/src/main.tsx), and [../web/frontend/src/components/layout/GameShell.tsx](../web/frontend/src/components/layout/GameShell.tsx). It also includes the sibling legacy path in [../web/frontend/src/components/GameView.tsx](../web/frontend/src/components/GameView.tsx), the dev-only harness in [../web/frontend/src/DevHarness.tsx](../web/frontend/src/DevHarness.tsx), component test files under [../web/frontend/src/components](../web/frontend/src/components), and the Django contract surface under [../web](../web).

### 5.4 Routes the real backend does not need to implement to replace the current mounted UI

The current mounted runtime will still render if these routes remain absent, because no mounted component calls them today: `/api/games/:id/summary/`, `/api/games/:id/timeseries/`, `/api/games/:id/economy/`, `/api/games/:id/communities/`, `/api/games/:id/organizations/`, `/api/games/:id/edges/`, `/api/games/:id/state-apparatus/`, `/api/games/:id/journal/`, `/api/games/:id/alerts/`, `/api/games/:id/node/:node_id/`, `/api/games/:id/org/:org_id/`, `/api/games/:id/community/:hyperedge_id/`, `/api/games/:id/edge/:edge_id/`, `/api/games/:id/hex/:h3_index/`, `/api/games/:id/actions/`, `/api/games/:id/actions/:action_id/`, `/api/games/:id/v2/world/`, and `/api/games/:id/v2/dialectics/:dialectic_id/`.

The current mounted runtime will not fully replace the MSW layer unless the backend provides compatibility for the three routes that the frontend actually calls in its present form and that do not align with the Django implementation: `/api/games/:id/actions/:verb/`, `/api/games/:id/actions/preview/` with flat `ActionPreviewResult` data, and `/api/games/:id/map/` if the store tests and legacy `GameView` path remain in scope.
