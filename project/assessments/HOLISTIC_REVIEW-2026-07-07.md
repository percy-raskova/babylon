## tests/ — architecture and hygiene (Python suite; static review, no suite run)

**Verdict:** This is a genuinely mature, large test estate (~767 test files, ~10,200 test functions, 39 conftests, a real Hypothesis invariant program with reusable harnesses, regression baselines, and disciplined xfail documentation) — but its enforcement spine is quietly broken: the `--strict-markers` flag the project believes is active (pyproject comment says "already have") is empirically a no-op under pytest 9.0.3, which is exactly why unregistered markers and a dead `ai` marker have crept in unnoticed. The `ai` marker is used by zero tests while both CLAUDE.md and the extended-analysis CI job select on it, so the AI eval lane selects nothing. TDD red_phase discipline has decayed into 30 permanently-RED tests (oldest from Dec 2025) excluded from CI but not from the local `mise` gate. Risk is moderate: the math/engine core is heavily and redundantly tested, but the guardrails that keep the taxonomy honest (strict markers, canary tests, property-test randomization) are each silently disarmed.

**Strengths:**
