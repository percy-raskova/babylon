# Phase 1 Data Model: Property-Based Tests for Conservation Invariants

**Date**: 2026-05-05
**Branch**: `053-conservation-invariants`

This feature does not introduce new domain entities or persistence schemas. The "data model" here is the set of test-side generator inputs and the test-side conservation predicates over existing domain types.

## 2. Existing types modified (additive marker only)

For every system class in `src/babylon/engine/systems/*.py` AND every substrate computer in `src/babylon/economics/substrate/{production,circulation,aggregation,equalization,ground_rent}.py`, add one class-level marker:

```python
class ExampleSystem:
    name: str = "example"
    creates_value: ClassVar[bool] = False  # NEW: default-deny opt-out marker
    ...
```

**Validation rule**: The c+v+s property test discovers systems via the existing system registry/protocol, reads the marker, and skips systems with `creates_value=True`. Systems without the marker are treated as `creates_value=False` (default-deny: included in the test).

**No state transitions, no persistence change, no schema migration.** The marker is read at test-collection time; it is not persisted, not serialized, and does not affect runtime behaviour.

Per the audit in `research.md` R1/R2, the seed values are:

| Class | Module | `creates_value` |
|-------|--------|-----------------|
| `ImperialRentSystem` | `engine/systems/economic.py` | True |
| `StruggleSystem` | `engine/systems/struggle.py` | True |
| `DispossessionEvents` | `engine/systems/dispossession_events.py` | True (default-deny while audit pending) |
| `DecompositionSystem` | `engine/systems/decomposition.py` | True (default-deny while audit pending) |
| `ProductionSystem` (engine-side) | `engine/systems/production.py` | False |
| All other engine systems in the canonical pipeline | `engine/systems/*.py` | False |
| `DefaultHexProductionComputer` (substrate-side) | `economics/substrate/production.py` | False (per spec; observes already-hydrated value) |
| `DefaultHexCirculationComputer` | `economics/substrate/circulation.py` | False (rescales row-stochastic OD; conservation by construction) |
| `DefaultResolutionAggregator` | `economics/substrate/aggregation.py` | False (sum aggregation; conservation by construction) |
| `DefaultHexEqualizationComputer` | `economics/substrate/equalization.py` | False (default-deny; flip to True if audit shows otherwise) |
| Other substrate computers (`ground_rent.*`, etc.) | `economics/substrate/*.py` | False (default-deny; flip to True if audit shows otherwise) |

## 3. New types introduced (test-side strategies only)

These live under `tests/property/strategies/` and are imported only by the new `tests/property/invariants/test_*.py` files. They are not part of the production `src/babylon/` package.

### 3.1 `HexGridStrategy`

```python
def hex_grid_strategy(
    min_hexes: int = 1,
    max_hexes: int = 25_000,
    *,
    seed_cells: Sequence[str] = MICHIGAN_RES7_SEED_CELLS,
) -> SearchStrategy[HexGrid]: ...
```

- Inputs: a pool of real H3 res-7 cell IDs covering Michigan (the canonical Article IV test case has ~50,000 res-7 cells statewide). The seed pool MUST be at least `max_hexes` cells; if smaller, the strategy clamps the draw to `min(requested_n, len(seed_cells))` and surfaces a warning.
- Output: a `HexGrid` with consistent res-6 and res-5 parent maps (computed via `h3.cell_to_parent`).
- Per-hex c, v, s drawn from `floats(min_value=0.0, max_value=1e6, allow_nan=False, allow_infinity=False)`.
- Hypothesis's default size-biased shrinking means most generated examples will be small (1–10 hexes); large examples (thousands of hexes) are exercised occasionally on default profile and exhaustively on the `slow` profile. This shape is intentional: small examples are fast to shrink on regression; large examples stress the scaled-tolerance `max(1e-10, 1e-11 * N)` at the Article IV scale.
- **Implementation note**: To populate `MICHIGAN_RES7_SEED_CELLS`, generate once at module-import time via `h3.polygon_to_cells(MICHIGAN_BOUNDARY_GEOJSON, res=7)` (Michigan boundary already available via Natural Earth SQLite per spec 036) and cache the result. Falls back to `WAYNE_OAKLAND_MACOMB_SEED_CELLS` if Natural Earth is unavailable in the test environment.

### 3.2 `ODMatrixStrategy`

```python
def od_matrix_strategy(
    n: int,
    *,
    density: float | None = None,
    flavor: Literal["identity", "empty_rows", "dense", "random"] = "random",
) -> SearchStrategy[scipy.sparse.csr_matrix]: ...
```

- Output: an `n × n` row-stochastic sparse CSR matrix (each non-zero row sums to 1.0).
- `flavor="identity"` — the identity matrix (no redistribution).
- `flavor="empty_rows"` — at least one zero-row hex.
- `flavor="dense"` — every row has multiple non-zero entries.
- `flavor="random"` — Bernoulli-mask + non-negative-weight random matrix.
- `n` typically matches the hex count of the paired `HexGrid` (so up to 25,000 in the upper bound). At large `n`, `density` SHOULD default to a low value (e.g., ≤0.01) to keep the sparse matrix tractable; commute matrices in real LODES data are extremely sparse (most pairs of cells have zero commute flow), so this matches the empirical regime.

### 3.3 `DPDStateGridStrategy`

```python
def dpd_state_grid_strategy(
    min_hexes: int = 1,
    max_hexes: int = 25_000,
) -> SearchStrategy[Mapping[str, DPDState]]: ...
```

- Output: a mapping of hex ID → `DPDState` with non-negative integer cohort counts (`integers(min_value=0, max_value=10_000)`).
- Same Hypothesis size-biased shrinking notes as §3.1 apply.

### 3.4 `CapitalStockTripleStrategy`

```python
def capital_stock_triple_strategy() -> SearchStrategy[tuple[float, float, float]]: ...
```

- Output: `(K_t, δ, I_t)` with `K_t ∈ [0, 1e9]`, `δ ∈ [0, 1]`, `I_t ∈ [0, 1e9]`.
- The `I_t` value is the "investment" term, which corresponds to the `c` (constant capital flow) argument of `DepreciationConfig.next_K(K_prev, c)`.

### 3.5 `WorldStateWithHexesStrategy`

```python
def worldstate_with_hexes_strategy(
    min_hexes: int = 1,
    max_hexes: int = 25_000,
) -> SearchStrategy[tuple[WorldState, HexGrid]]: ...
```

- Composes the existing `worldstate_strategy()` (spec 040) with `hex_grid_strategy()`.
- Output: a tuple `(WorldState, HexGrid)`. The two are independent state containers (no shared node IDs); each is passed to its appropriate consumer (engine systems take the graph derived from `WorldState.to_graph()`; substrate computers take `HexGrid` directly).
- Lives in `tests/property/strategies/worldstate.py` (extends the existing spec-040 file).
- **Resolves analyze finding U1**.

### 3.6 Test fixtures (in `tests/property/conftest.py`)

```python
@pytest.fixture
def service_container_fixture() -> ServiceContainer: ...

@pytest.fixture
def tick_context_fixture() -> TickContext: ...
```

- `service_container_fixture`: minimal `ServiceContainer` with default `defines`, `formulas`, an in-memory `event_bus`, and a stub `database` sufficient for a single tick.
- `tick_context_fixture`: minimal `TickContext(tick=0)` (or `dict(tick=0)` for legacy compatibility per `engine.systems.protocol.ContextType`).
- **Resolves analyze finding U2** for full-pipeline tests.

## 4. Conservation predicates (test-side)

Each predicate is a Python function used by exactly one `@given` test. They are not exported from `src/`.

| Predicate | Signature | Tolerance | Used by |
|-----------|-----------|-----------|---------|
| `cvs_conserved_substrate` | `(pre: HexGrid, post: HexGrid) -> bool` | `tol(N) = max(1e-10, 1e-11 * N)` | INV-001 per-substrate-computer test (T019) |
| `hex_unchanged_engine` | `(pre: HexGrid, post: HexGrid) -> bool` | exact (`==`) on every hex c/v/s field | INV-001 per-engine-system test (T019a) |
| `cvs_conserved_pipeline` | `(pre_hex: HexGrid, post_hex: HexGrid, recorded_deltas: dict) -> bool` | `tol(N)` net of `creates_value=True` recorded deltas | INV-001 full-pipeline test (T020) |
| `h3_sheaf_glued` | `(grid: HexGrid, target_resolution: int) -> bool` | `1e-10` | INV-002 (T024–T026) |
| `circulation_v_conserved` | `(pre: HexGrid, post: HexGrid) -> bool` | `tol(N) = max(1e-10, 1e-11 * N)` | INV-003 (T031–T033) |
| `population_accounting_holds` | `(pre_pop: int, post_pop: int, births: int, deaths: int) -> bool` | exact (`==`) | INV-004 (T037, T038) |

## 5. Counterexample database

The Hypothesis example database lives at `.hypothesis/examples/` (default location). It is not committed to git but persists across local runs and is preserved by the CI cache. No schema; the database is opaque to the test code.

## 6. Out of scope for this data model

- New runtime types in `src/babylon/`.
- New persistence tables.
- New JSON contracts for any external API.
- Changes to existing type validation rules in `HexGrid`, `WorldState`, or `DPDState`.

The five conservation predicates are testing the existing types' invariance under the existing engine; they do not extend the type system itself.
