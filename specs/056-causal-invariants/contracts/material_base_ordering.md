# Contract: Material Base Ordering (US1, INV-013)

**Invariant ID**: INV-013
**User Story**: US1 (P1) — Material Base runs before Action Phase
**Constitutional Basis**: ADR032 Materialist Causality; Constitution I.18 Material-Ideological Distinction; III.7 Determinism
**Test File**: `tests/property/invariants/test_material_base_ordering.py`

**Pre-test prerequisite (F6=α, 2026-05-07)**: This test depends on the
Phase 2 reorder of `_DEFAULT_SYSTEMS` so `OODASystem` runs at
position 14 (immediately after `MetabolismSystem`, immediately before
`SurvivalSystem`). The current codebase orders OODA at position 21;
without the reorder, this test trivially passes (every Material Base
System trivially precedes the last-position OODA), making the
invariant vacuous as a regression guard. The reorder is part of T004's
production work.

## Predicate

```text
For every full SimulationEngine.run_tick() invocation:
  For every system M in MATERIAL_BASE_SYSTEMS:
    For every system A in ACTION_PHASE_SYSTEMS:
      If both M and A are in `engine.systems` AND both were invoked this tick:
        call_index(M) < call_index(A)
```

In plain English: every Material Base System must complete its `step()`
call **before** any Action Phase System (currently `OODASystem`) begins
its `step()` call. The check is per-tick; ordering across ticks is
unconstrained (correct: each tick re-runs the whole pipeline).

## Inputs

- `SimulationEngine` constructed with the default System list (or a test
  alternate list)
- A random `WorldState` generated by `worldstate_strategy()` from Spec 040
- A `ServiceContainer.testing()` instance
- A `TickContext` with `tick=N` for an arbitrary N

## Outputs

- A `TickTrace` containing `system_calls: tuple[SystemCallEvent, ...]`
- For each Material Base System invoked this tick: a `SystemCallEvent`
  recorded in `system_calls` BEFORE the `OODASystem` event

## Falsification

The test fails if any of the following hold:

| Condition | Failure Mode |
|-----------|--------------|
| `call_index(M) >= call_index(A)` for any M ∈ Material Base, A ∈ Action Phase | Inversion detected; failure message names `(material_base_system, action_phase_system)` and the inverted call indices |
| `OODASystem` was invoked more than once per tick | Acceptance Scenario 3 violation; failure message names the duplicate call indices |
| The `MATERIAL_BASE_SYSTEMS` and `ACTION_PHASE_SYSTEMS` constants do not partition `_DEFAULT_SYSTEMS` | Import-time `AssertionError` from `simulation_engine.py`; test never runs |

## Implementation Notes

- The Material Base / Action Phase / Consequence sets are imported directly
  from `babylon.engine.simulation_engine` — no hand-maintained list in the
  test module (FR-002).
- `SystemCallSpy` wraps every System.step in `engine.systems`, recording
  `(system_class_name, call_index, monotonic_timestamp_ns)` per
  invocation (FR-003).
- The spy is observably non-interfering: a separate test
  `test_spy_does_not_alter_post_state` runs the same starting state with
  and without the spy, asserts post-tick `model_dump`-equality under
  Spec 055 exclude rules (FR-012).
- Acceptance Scenario 2 is exercised via a deliberately-permuted System
  list — the spy detects the inversion and the test fails with a minimal
  example.
- Acceptance Scenario 3 is checked by counting OODA invocations per tick
  trace; > 1 fails immediately.

## Default-Deny + Opt-Out Marker

A System that legitimately needs to be invoked twice per tick (none
currently exist) MUST flag itself with:

```python
class SomeSystem:
    bypasses_causal_invariant: ClassVar[dict[str, str]] = {
        "single_invocation_per_tick": "Justification text here.",
    }
```

The harness consumes this marker, skips the named predicate for the
flagged System, and machine-asserts that the value is non-empty (FR-011).
